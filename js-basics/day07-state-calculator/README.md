# Day07 - 상태 기반 계산기 리팩토링

## 프로젝트 개요
이 프로젝트는 사용자로부터 가격과 수량을 입력받아 총액을 계산하고, 그 결과를 화면에 표시하는 계산기입니다.  
입력값에 대한 검증을 거쳐, 잘못된 입력에는 해당하는 오류 메시지를 표시합니다.  
**`state`** 객체를 사용하여 상태를 관리하고, **`switch-case`** 문을 이용하여 사용자에게 적절한 메시지를 출력하는 방식으로 구현되었습니다.

## 요구사항
- 사용자는 가격과 수량을 입력합니다.
- 가격 또는 수량이 빈값, 0 이하, 숫자가 아닌 경우 적절한 오류 메시지를 출력합니다.
- 정상적인 값이 입력되면 총액을 계산하여 결과를 표시합니다.

## 프로젝트 목표
1. **상태 관리**: 입력값과 상태를 `state` 객체로 통합하여 관리
2. **입력 검증**: 가격과 수량을 검증하고, 상태에 따라 오류 메시지를 출력
3. **UI 업데이트**: 상태를 기반으로 화면을 업데이트하는 방식 구현

## 기능
1. **입력값 검증**:
   - 빈값, 숫자 아닌 값, 0 이하의 값에 대한 검증
2. **결과 메시지**:
   - 가격과 수량을 올바르게 입력 시 총액을 계산하여 결과를 표시
   - 잘못된 값 입력 시 오류 메시지를 출력
3. **상태 관리**:
   - 상태(state) 객체를 이용해 **price**, **qty**, **status**, **message**를 관리

## 사용법
1. **가격**과 **수량**을 입력합니다.
2. **계산하기** 버튼을 클릭하여 결과를 확인합니다.
3. 올바른 입력값을 제공하면 총액을 계산하여 결과를 표시합니다.
4. 잘못된 값이 입력되면 오류 메시지가 표시됩니다.
